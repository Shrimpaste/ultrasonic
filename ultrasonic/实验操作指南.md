# 超声波测距模块对比实验 — 详细操作指南

> 本文档为实验的完整操作手册，涵盖从环境搭建到数据分析的全流程，适合首次接触 ESP-IDF 和超声波传感器的开发者。

---

## 一、实验目标与原理

### 1.1 实验目标

系统比较两款超声波测距模块在**精度、稳定性、抗干扰性、角度响应**等维度的性能差异：

| 模块 | 通信方式 | 分辨率 | 量程 | 特色 |
|---|---|---|---|---|
| **A02YYUW** | UART 9600bps | **1 mm** | 30~4500 mm | 防水探头，抗光照干扰 |
| **URM09 (SEN0304)** | I2C 0x11 | **1 cm** | 20~5000 mm | 内置温度补偿 |

### 1.2 超声波测距原理

```
发射超声波脉冲 → 遇到目标反射 → 接收回波 → 计算飞行时间

距离 = (声速 × 飞行时间) / 2
```

- 声速约 **340 m/s**，温度每升高 1°C 约增加 0.6 m/s
- 除以 2 是因为声波经历「发射 → 目标 → 返回」两段路程
- **URM09 内置温度传感器**，可实时修正声速，提高精度
- **A02YYUW 采用双探头设计**（TX/RX 分离），盲区可压缩至 3 cm

### 1.3 两模块通信协议

#### A02YYUW — UART 被动输出

模块以约 10Hz 频率持续输出 4 字节帧，主控只需接收：

```
帧格式：[0xFF] [H] [L] [SUM]
距离(mm) = (H << 8) | L
校验和   = (0xFF + H + L) & 0xFF
有效范围：30 ~ 4500 mm，返回 0 表示超出量程
```

**驱动实现要点**（[a02yyuw.c](main/a02yyuw.c)）：
1. 初始化 UART1，9600bps，RX=GPIO18，TX=GPIO17
2. 读取时先 flush 旧数据，等 100ms 收集新帧
3. 批量读取缓冲区，扫描 `0xFF` 帧头，取最后一帧（最新数据）
4. 验证校验和，解析距离值（单位 mm）

#### URM09 — I2C 寄存器读写（V3.0 版本）

主控主动触发测量，寄存器映射：

| 寄存器 | 地址 | 说明 |
|---|---|---|
| CMD | 0x08 | 写 0x01 触发一次测量 |
| DIST_H/L | 0x03/0x04 | 距离高/低字节 (cm) |
| TEMP_H/L | 0x05/0x06 | 温度高/低字节 (×0.1°C, 有符号) |

> **注意**：URM09 V3.0 版本的寄存器地址与早期版本不同。CMD 寄存器为 0x08（非 0x01），量程配置寄存器 0x11 在新版本中已无效。

**驱动实现要点**（[urm09.c](main/urm09.c)）：
1. 使用 ESP-IDF v6.x 新版 `i2c_master` API
2. 初始化 I2C0，SDA=GPIO8，SCL=GPIO9，设备地址 0x11
3. 写 CMD 寄存器（0x08）触发测量，等待 **120ms**
4. 读取距离（0x03）+温度（0x05），共 4 字节
5. 自动过滤异常值：0xFFFF 和超量程值

## 二、硬件准备与接线

### 2.1 物料清单

| 编号 | 器件 | 数量 | 备注 |
|---|---|---|---|
| 1 | ESP32-S3 开发板 | 1 | 已配置 IDF 环境 |
| 2 | A02YYUW 超声波模块 | 1 | UART，4线（VCC/GND/TX/RX） |
| 3 | DFRobot URM09 (SEN0304) | 1 | I2C，4线（VCC/GND/SDA/SCL） |
| 4 | 4.7 kΩ 电阻 | 2 | I2C 上拉（URM09 板载可能已有） |
| 5 | 面包板 + 杜邦线 | 若干 | — |
| 6 | USB 数据线 | 1 | 烧录 + 串口监视 |
| 7 | 钢卷尺 / 激光尺 | 1 | 标定参考距离 |
| 8 | 平整反射板（A4 纸/木板） | 1 | 作为测距目标物 |

### 2.2 接线方案

#### A02YYUW → ESP32-S3

```
A02YYUW 红线 (VCC)      →  3.3V
A02YYUW 黑线 (GND)      →  GND
A02YYUW 绿线/黄线 (TX)  →  GPIO18  ← UART1 RXD（传感器发送→ESP32接收）
A02YYUW 白线 (RX)       →  GPIO17  ← UART1 TXD（保持 HIGH=稳定模式）
```

#### URM09 → ESP32-S3

```
URM09 VCC  →  3.3V
URM09 GND  →  GND
URM09 SDA  →  GPIO8   ← I2C0 SDA（外接 4.7kΩ 上拉至 3.3V）
URM09 SCL  →  GPIO9   ← I2C0 SCL（外接 4.7kΩ 上拉至 3.3V）
```

#### GPIO 分配总览

| 模块 | 信号 | GPIO | 说明 |
|---|---|---|---|
| A02YYUW | UART1 RX | **GPIO18** | 传感器 TX（绿/黄线）→ ESP32 RX |
| A02YYUW | UART1 TX | **GPIO17** | 传感器 RX（白线），保持 HIGH |
| URM09 | I2C SDA | **GPIO8** | D 线，需上拉 4.7kΩ |
| URM09 | I2C SCL | **GPIO9** | C 线，需上拉 4.7kΩ |

> **注意**：ESP32-S3 的 GPIO17 是 UART1 TX（发送），GPIO18 是 UART1 RX（接收）。传感器 TX 必须接 ESP32 的 RX。

### 2.3 接线检查清单

- [ ] 两模块供电均为 3.3V（勿接 5V，ESP32-S3 IO 不耐 5V）
- [ ] GND 共地
- [ ] A02YYUW 的 TX（绿/黄线）接 ESP32 的 **GPIO18**（U1RXD）
- [ ] A02YYUW 的 RX（白线）接 ESP32 的 **GPIO17**（U1TXD）
- [ ] URM09 的 D(SDA) 接 GPIO8，C(SCL) 接 GPIO9
- [ ] URM09 的 SDA/SCL 已接上拉电阻（若模块板载没有）
- [ ] 面包板接触良好，无松动

## 四、测试阶段详解

程序按顺序执行 **7 个测试阶段**，每阶段采集完成后输出统计数据（JSON + 人类可读摘要），然后**暂停等待用户按 ENTER** 继续下一阶段。这样可以在阶段间从容调整反射板位置和角度。

### 阶段总览

| 阶段 | 编号 | 名称 | 目标距离 | 采样数 | 考察指标 |
|---|---|---|---|---|---|
| 0 | Static_20cm | 20 cm | 30 | 盲区、近距精度 |
| 1 | Static_100cm | 100 cm | 30 | 常用距离精度 |
| 2 | Static_300cm | 300 cm | 30 | 远距衰减、精度退化 |
| 3 | Angle_0deg | 20 cm | 30 | 基准正对 |
| 4 | Angle_30deg | 20 cm | 30 | 轻度偏转 |
| 5 | Angle_45deg | 20 cm | 30 | 边缘波束角 |
| 6 | Stability_100x | 50 cm | 100 | 噪声分布、漂移 |

### 阶段 0 — 静态近距（Static_20cm）

**目的**：测试两模块在近距（接近盲区）时的表现

**操作步骤**：
1. 将反射板竖直放置，用卷尺量出距两探头 **20.0 cm** 处做标记
2. 两模块探头**正对**反射板，水平放置，保持静止
3. 等待串口输出 `phase_start`，程序自动采集 30 次
4. 观察是否出现"测量失败"或异常值

**预期结果**：
- A02YYUW：盲区 3cm，20cm 应在有效范围内，精度 ±3mm
- URM09：盲区 ~2cm，20cm 应在有效范围内，精度 ±1cm

**关键观察**：哪个模块在近距时失败率更高？

### 阶段 1 — 静态中距（Static_100cm）

**目的**：测试两模块在常用工作距离的精度

**操作步骤**：
1. 反射板移至 **100.0 cm** 处
2. 确保测试环境无运动物体干扰
3. 程序自动采集 30 次

**预期结果**：
- 此距离为两模块标称最佳工作区间
- 两者读数应高度一致
- A02YYUW 标准差应 < 5mm，URM09 标准差应 < 2cm

### 阶段 2 — 静态远距（Static_300cm）

**目的**：测试远距离时的精度退化和信号衰减

**操作步骤**：
1. 反射板移至 **300.0 cm** 处
2. 确保走廊或空旷环境，无侧面反射干扰
3. URM09 已自动配置为 500cm 量程

**预期结果**：
- A02YYUW（1mm 分辨率）噪声量级应更小
- URM09（1cm 分辨率）可能有更大的量化噪声
- 远距时信号衰减，两模块失败率可能上升

**注意事项**：
- 确保探头 ±30° 范围内无杂物
- 避免在狭窄走廊测试（侧壁反射会干扰）

### 阶段 3~5 — 角度测试（Angle_0/30/45deg）

**目的**：测试探头在非正对角度时的测量能力

**操作步骤**：
1. 反射板固定于 **20 cm**，保持不动
2. **转动探头**（不是反射板），相对反射板法线偏转：
   - 阶段 3：0°（基准正对）
   - 阶段 4：30°（轻度偏转）
   - 阶段 5：45°（边缘波束角）
3. 使用量角器确认角度
4. 重点观察：哪个模块在大角度时最先出现大量无效读数

**预期结果**：
- 0°：两者均正常
- 30°：A02YYUW 可能开始出现少量失败
- 45°：接近波束角边缘，可能有较多失败
- URM09 波束角可能更窄（±15°），大角度时失败率更高

**关键观察**：记录每个角度下的失败率，绘制角度-失败率曲线

### 阶段 6 — 稳定性（Stability_100x）

**目的**：评估读数噪声形态和长期稳定性

**操作步骤**：
1. 反射板固定于 **50 cm** 正对，**绝对静置**
2. 连续采集 **100 次**
3. 程序自动计算均值/标准差/最大偏差

**预期结果**：
- 分布应接近高斯分布（单峰）
- 若出现双峰，说明环境中有第二反射面
- A02YYUW 标准差应更小（1mm 分辨率）
- URM09 温度读数应稳定（室内温度变化小）

**关键观察**：
- 绘制距离分布直方图
- 检查是否有漂移（均值是否随时间变化）
- 检查是否有异常值（超出 3σ 的点）

### 5.1 三种 JSON 行类型

```json

```

| 字段 | 含义 | 单位 |
|---|---|---|
| `t` | 启动后毫秒时间戳 | ms |
| `ph` | 测试阶段编号（-1=连续监测） | — |
| `i` | 阶段内采样序号 | — |

| `a.ok` | A02YYUW 本次读取是否有效 | bool |
| `u.d` | URM09 距离 | **cm** |
| `u.ok` | URM09 本次读取是否有效 | bool |
| `u.temp` | URM09 内置温度 | °C |

```json
{"event":"phase_stats","phase":1,"name":"Static_100cm",
```

## 八、数据分析方法

### 8.1 关键指标计算

#### 精度评估

```
绝对误差 = |测量均值 - 真实距离|
相对误差 = 绝对误差 / 真实距离 × 100%
```

#### 稳定性评估

```
标准差 σ = √(Σ(xi - x̄)² / (n-1))
变异系数 CV = σ / x̄ × 100%    （越小越稳定）
最大偏差 = max(|xi - x̄|)
```

#### 一致性评估

```
Pearson 相关系数 r = Σ((xi-x̄)(yi-ȳ)) / √(Σ(xi-x̄)² × Σ(yi-ȳ)²)
r > 0.99：高度一致
r > 0.95：良好一致
r < 0.90：存在差异
```

### 8.2 预期性能差异

| 测试维度 | 预期优势方 | 原因分析 |
|---|---|---|
| 分辨率/近距精度 | **A02YYUW** | 1mm vs 1cm 硬件分辨率 |
| 远距稳定性（温变环境） | **URM09** | 内置温度补偿修正声速 |
| 通信可靠性 | **URM09** | I2C 应答机制 vs UART 单向广播 |
| 大角度测量（45°） | 待验证 | 取决于各自波束角宽度 |
| 抗强光干扰 | **A02YYUW** | 产品页明确标注不受光照影响 |
| 综合性价比 | **A02YYUW** | ¥65 vs ¥89，防水加分 |

### 8.3 数据异常排查

| 现象 | 可能原因 | 排查方法 |
|---|---|---|
| A02YYUW 持续返回失败 | UART RX 接错引脚 / 波特率不对 | 用示波器或逻辑分析仪抓 GPIO18 波形 |
| URM09 I2C 初始化失败 | 上拉电阻缺失 / 地址冲突 | 运行 `i2c_scan` 扫描总线确认 0x11 存在 |
| 两模块读数相差 >5 cm | 探头未对齐 / 一模块角度偏移 | 用直角尺确认两模块探头平行且正对目标 |
| 稳定性测试出现双峰分布 | 环境中有第二反射面（墙/桌边） | 清空测试区域，确保探头 ±30° 范围内无杂物 |
| URM09 温度读数异常 | 寄存器字节序理解有误 | 温度为有符号 int16，单位 0.1°C，需强制转换 |

```c
// 初始化 UART1，9600bps，RX=指定引脚
esp_err_t a02yyuw_init(int rx_pin);
```

```c
// 初始化 I2C0 总线 + 添加 0x11 设备（V3.0 版本）
esp_err_t urm09_init(i2c_master_bus_handle_t *bus,
                     i2c_master_dev_handle_t *dev,
                     int sda_pin, int scl_pin);

// 触发测量（写 CMD=0x08 寄存器）→ 等待 120ms → 读距离(0x03)+温度(0x05)

esp_err_t urm09_read(i2c_master_dev_handle_t dev, urm09_result_t *result);
```

### 9.3 主程序流程（[main.c](main/main.c)）

```
app_main()
  ├── a02yyuw_init(GPIO18, GPIO17)   # 初始化 A02YYUW UART (RX=18, TX=17)
  ├── urm09_init(GPIO8, GPIO9)       # 初始化 URM09 I2C
  │
  └── for (phase = 0; phase < 7; phase++)   # 遍历 7 个测试阶段
        ├── emit_phase_start()               # 输出阶段开始事件
        │
        └── for (i = 0; i < samples; i++)   # 每阶段采集 N 次
              ├── a02yyuw_read(&a_result)    # 先读 A02YYUW
              ├── urm09_read(dev, &u_result) # 再读 URM09
              ├── emit_sample()              # 输出 JSON 采样行
              └── vTaskDelay(50ms)           # 间隔 50ms
        │
        ├── compute_stats()                  # 计算统计量
        ├── emit_phase_stats()               # 输出阶段统计
        └── vTaskDelay(3s)                   # 阶段间等待
  │
  └── emit_test_complete()                   # 输出测试完成事件
```

### 9.4 关键设计决策

1. **UART 帧同步**：A02YYUW 以 ~10Hz 持续输出，驱动在读取时丢弃不完整帧直到找到 `0xFF` 帧头，超时 1 秒
2. **I2C 时序**：URM09 触发后需等待 120ms（500cm 量程），使用 `vTaskDelay(pdMS_TO_TICKS(120))`
3. **采样间隔**：每对读数之间间隔 50ms，避免传感器过载
4. **JSON 输出格式**：严格按实验规划文档定义的格式，A02 单位 mm，URM09 单位 cm
5. **错误处理**：读取失败记为 -1，统计时排除失败值，单独记录失败计数

## 十、常见问题 FAQ

### Q1：编译报错 `unknown type name 'gpio_num_t'`

**原因**：ESP-IDF v6.x 中 GPIO 类型定义路径变化

**解决**：项目已使用 `int` 替代 `gpio_num_t`，无需额外操作

### Q2：URM09 I2C 初始化失败

**排查步骤**：
1. 检查接线：SDA→GPIO8，SCL→GPIO9
2. 检查上拉电阻：是否有 4.7kΩ 上拉至 3.3V
3. 扫描 I2C 总线：在代码中添加 `i2c_master_probe()` 确认设备存在
4. 检查供电：确保 3.3V 供电稳定

### Q3：A02YYUW 读取超时

**排查步骤**：
1. 检查 TX/RX 是否交叉连接（模块 TX → ESP32 RX）
2. 确认波特率 9600bps
3. 用示波器检查 GPIO18 是否有信号

### Q4：两模块读数差异大

**排查步骤**：
1. 确认探头平行且正对目标
2. 检查目标物是否平整（粗糙表面会散射超声波）
3. 确认目标距离在两模块量程内
4. 检查是否有侧面反射干扰

### Q5：稳定性测试出现双峰分布

**原因**：探头 ±30° 范围内有第二反射面（墙、桌边等）

**解决**：清空测试区域，确保探头正前方无障碍物

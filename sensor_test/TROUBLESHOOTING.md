# 超声波传感器调试记录

## 最终接线

```
A02YYUW (型号: A0221AU5CL0895)    ESP32-S3
─────────────────────────        ──────────
红线 (VCC)           ───→        3.3V
黑线 (GND)           ───→        GND
白线 (传感器 TX)     ───→        GPIO18 (U1RXD)
黄线 (传感器 RX)     ───→        GPIO17 (U1TXD)

URM09 (V3.0)                     ESP32-S3
─────────────                    ──────────
VCC                  ───→        3.3V
GND                  ───→        GND
D (SDA)              ───→        GPIO8  (需外接 4.7kΩ 上拉至 3.3V)
C (SCL)              ───→        GPIO9  (需外接 4.7kΩ 上拉至 3.3V)
```

> **重要**：ESP32-S3 的 GPIO17 是 U1TXD（发送），GPIO18 是 U1RXD（接收）。传感器 TX 接 ESP32 RX。

## 调试过程中发现的问题与修复

### 问题 1：A02YYUW 线序接反

| 项目 | 说明 |
|------|------|
| **现象** | GPIO18 恒 LOW (0V)，无任何 UART 数据 |
| **原因** | 黄线不是 TX，白线才是传感器 TX |
| **验证方法** | 诊断程序读 GPIO 电平：恒 HIGH = UART 空闲态 = TX 引脚 |
| **修复** | 白线 → GPIO18（RX），黄线 → GPIO17（TX） |

### 问题 2：URM09 寄存器地址错误

| 项目 | 说明 |
|------|------|
| **现象** | URM09 始终返回 8448 cm / 0.1°C |
| **原因** | 代码用的寄存器地址与 V3.0 硬件不符 |
| **官方寄存器表（V3.0）** | |

| 寄存器 | 地址 | 说明 |
|--------|------|------|
| 设备地址 | 0x00 | 默认 0x11 |
| 产品 ID | 0x01 | 固定 0x01 |
| 版本号 | 0x02 | 0x10 = V1.0 |
| 距离高位 | **0x03** | 单位 cm |
| 距离低位 | **0x04** | |
| 温度高位 | **0x05** | ÷10 得 °C |
| 温度低位 | **0x06** | |
| 配置寄存器 | 0x07 | bit7: 0=被动 1=自动 |
| 命令寄存器 | **0x08** | 写 0x01 触发测距 |

| 修复 | 说明 |
|------|------|
| 命令寄存器 | 0x01 → **0x08** |
| 读取起始地址 | 0x02 → **0x03** |
| 量程配置 | 新版本已取消，无需设置 |

### 问题 3：URM09 dist=0 误判为错误

| 项目 | 说明 |
|------|------|
| **现象** | 距离为 0 时显示 FAIL |
| **原因** | 代码把 dist==0 当作 ESP_ERR_NOT_FOUND |
| **修复** | dist=0 表示无障碍物（超出量程），是正常值 |

### 问题 4：URM09 距离异常值

| 项目 | 说明 |
|------|------|
| **现象** | 偶尔出现 65535 cm 等异常值 |
| **原因** | 传感器错误码或 I2C 总线噪声 |
| **修复** | 过滤 0xFFFF 和 >500 cm 的值 |

### 问题 5：A02YYUW 帧格式错误

| 项目 | 说明 |
|------|------|
| **现象** | 校验始终失败 |
| **原因** | 最初误用双 0xFF 帧头 + 7字节帧格式 |
| **正确格式** | 4字节：`[0xFF] [DATA_H] [DATA_L] [SUM]` |
| **校验和** | `SUM = (0xFF + DATA_H + DATA_L) & 0xFF` |

### 问题 6：A02YYUW 数据不稳定

| 项目 | 说明 |
|------|------|
| **现象** | 大量校验失败，偶尔有正确帧 |
| **原因** | GND 接触不良导致信号噪声 |
| **软件缓解** | 保留上次有效值 |
| **硬件解决** | 检查/重新焊接 GND 焊点 |

### 问题 8：A02YYUW 读取延迟大

| 项目 | 说明 |
|------|------|
| **现象** | A02 数据更新明显慢于 URM09 |
| **原因** | 逐字节读取（每字节一次 syscall）+ 600ms 超时 + 3 次重试 |
| **旧流程** | 找 0xFF（逐字节，每字节等 20ms）→ 读 3 字节 → 失败重试 ×3 |
| **最坏耗时** | 3 × 600ms + 200ms 循环延迟 ≈ 2 秒 |
| **修复** | 改为批量读取 + 缓冲区扫描 |
| **新流程** | flush 清旧数据 → 等 100ms → 批量读缓冲区 → 扫描取最新帧 |
| **新耗时** | 100ms + 几毫秒 ≈ **100ms** |

### 问题 7：GPIO 配置干扰 UART

| 项目 | 说明 |
|------|------|
| **现象** | 诊断函数调用 gpio_set_direction 后 UART 停止工作 |
| **原因** | gpio_set_direction 会覆盖 UART IO MUX 配置 |
| **修复** | 诊断函数只用 uart_read_bytes，不操作 GPIO |

## I2C 上拉电阻

URM09 必须在 SDA/SCL 上各接 **4.7kΩ 上拉电阻至 3.3V**。ESP32 内部上拉不够强，不接会导致数据位被噪声翻转，距离值随机跳变。

---

## 诊断功能

程序启动时会自动运行两段诊断：

### A02 UART 诊断
- 清空 UART 缓冲区，等 800ms 抓原始字节
- 尝试解析 4 字节帧（0xFF 帧头 + 校验）
- 显示收到的原始 HEX 数据和帧解析结果

### URM09 I2C 诊断
- 扫描 I2C 总线上的所有设备
- 逐寄存器 dump（0x00-0x15）
- 对比触发测量前后的寄存器值

## A02 读取优化细节

### 旧方案（逐字节读取）

```
循环 {
  uart_read_bytes(1字节)  ← syscall 开销
  检查是否 0xFF
  超时 600ms
}
重复 3 次
```

问题：每次 syscall 有 FreeRTOS 上下文切换开销，逐字节效率低。

### 新方案（批量读取 + 扫描）

```
uart_flush_input()     ← 清空旧数据
vTaskDelay(100ms)      ← 等传感器发一帧
uart_read_bytes(64字节) ← 一次读完缓冲区
扫描 buf[] 取最新有效帧 ← 纯内存操作
```

优势：
- 只有 1 次 syscall（读取），而非 N 次
- 无重试逻辑
- 取最新帧而非最早帧，响应更快
- 传感器每 100ms 发一帧，100ms 等待刚好匹配

## 时序分析

| 组件 | 刷新率 | 单次读取耗时 |
|------|--------|-------------|
| A02YYUW | ~10 Hz (100ms/帧) | ~100ms（等待+读取） |
| URM09 | ~8 Hz (120ms/次) | ~120ms（触发+等待+读取） |
| 主循环 | ~5 Hz | 取较慢者 + 100ms 延迟 |

## ESP32-S3 UART 引脚对照（引脚定义表）

| 引脚 | GPIO | 功能 |
|------|------|------|
| IO17 | GPIO17 | U1TXD（UART1 发送） |
| IO18 | GPIO18 | U1RXD（UART1 接收） |
| TXD0 | GPIO43 | U0TXD（USB 串口发送） |
| RXD0 | GPIO44 | U0RXD（USB 串口接收） |

> GPIO43/44 被 USB 串口占用，不能分配给传感器。

# 超声波传感器硬件调试工具

在正式实验之前，用此例程快速验证 A02YYUW 和 URM09 两个超声波模块是否正常工作。

---

## 功能

- 同时初始化 A02YYUW（UART）和 URM09（I2C）两个传感器
- 持续以 ~5Hz 频率读取并输出格式化表格
- 每个传感器独立显示距离、温度、状态（✓ OK / ✗ FAIL）
- 初始化时自动检测硬件连接，失败会给出明确的接线提示

串口输出效果示例：

```

║  序号  ║    A02YYUW (UART)     ║      URM09 (I2C)         ║
║        ║  距离(mm)    状态     ║  距离(cm)  温度(°C) 状态  ║
╠════════╬═══════════════════════╬═══════════════════════════╣
║    1   ║    198 mm    ✓ OK    ║    20 cm   24.3°C  ✓ OK  ║

║    3   ║     ---      ✗ FAIL  ║    ---     ---°C  ✗ FAIL  ║
```

## 接线

```
A02YYUW 模块              ESP32-S3
─────────────            ──────────
红线 (VCC)      ───→     3.3V
黑线 (GND)      ───→     GND
绿线/黄线 (TX)  ───→     GPIO18     ← 注意：模块 TX 接 ESP32 的 RX (U1RXD)
白线 (RX)       ───→     不接

URM09 模块                ESP32-S3
─────────────            ──────────
VCC            ───→     3.3V
GND            ───→     GND
D  (SDA)       ───→     GPIO8      ← 需外接 4.7kΩ 上拉至 3.3V
C  (SCL)       ───→     GPIO9      ← 需外接 4.7kΩ 上拉至 3.3V
```

> 两模块可同时连接，UART 与 I2C 总线互不干扰。

## 使用方法

### 1. 编译烧录

```bash
cd /home/sp/auto-box/sensor_test

# 激活 IDF 环境（如已写入 .bashrc 可跳过）
source ~/esp/esp-idf/export.sh

# 首次需设置目标芯片
idf.py set-target esp32s3

# 编译
idf.py build

# 烧录并打开串口监视
idf.py flash monitor
```

### 2. 观察输出

烧录成功后串口会显示：

```
┌─────────────────────────────────────────────────┐
│     超声波传感器硬件调试工具  v1.0               │
│     A02YYUW (UART) + URM09 (I2C)               │
└─────────────────────────────────────────────────┘

  A02YYUW OK — UART1 RX=GPIO18
  URM09  OK — I2C0 SDA=GPIO8 SCL=GPIO9 addr=0x11

  ✓ 两模块初始化成功，开始持续读取...
```

随后开始输出实时读数表格。

### 3. 验证方法

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 保持探头前方空旷（>1m） | 两模块显示较大距离值，状态均为 ✓ OK |
| 2 | 手掌靠近探头（~20cm） | 距离值明显减小，两模块读数接近 |
| 3 | 手掌移开 | 距离值恢复 |
| 4 | 遮挡其中一个探头 | 被遮挡模块显示 ✗ FAIL 或距离异常，另一个正常 |

### 4. 退出

```
Ctrl + ]
```

## 故障排查

### A02YYUW 显示 ✗ FAIL

| 可能原因 | 排查方法 |
|---|---|
| TX/RX 接反 | 确认模块 TX（绿线/黄线）→ ESP32 **GPIO18**（U1RXD） |
| GND 接触不良 | 检查黑线两端是否插紧，尝试焊接固定 |
| 供电问题 | 用万用表确认模块 VCC 为 3.3V |
| 模块损坏 | 换一个 A02YYUW 测试 |

> **注意**：ESP32-S3 的 GPIO17 是 UART1 TX（发送），GPIO18 才是 UART1 RX（接收）。传感器 TX 必须接 ESP32 的 RX。

### URM09 显示 ✗ FAIL 或初始化失败

| 可能原因 | 排查方法 |
|---|---|
| 上拉电阻缺失 | 在 SDA、SCL 线上各接一个 4.7kΩ 电阻到 3.3V |
| SDA/SCL 接反 | 确认模块 D→SDA→GPIO8，C→SCL→GPIO9 |
| I2C 地址冲突 | 扫描总线确认设备地址为 0x11 |
| 模块损坏 | 换一个 URM09 测试 |

> **URM09 V3.0 寄存器说明**：命令寄存器为 0x08（写 0x01 触发测距），距离在 0x03-0x04，温度在 0x05-0x06。量程配置（0x11）在新版本中已无效。

### 两个都 FAIL

| 可能原因 | 排查方法 |
|---|---|
| ESP32 未正常启动 | 检查 USB 连接，按 RST 复位 |
| 串口波特率不对 | 确认监视器波特率为 115200 |
| 供电不足 | 避免同时从 USB 取电给多个外设 |

## 与正式实验项目的关系

```
auto-box/
├── sensor_test/      ← 本项目：硬件调试，验证传感器可用
│   └── main.c           单文件，无外部驱动依赖
│
└── ultrasonic/       ← 正式实验：7阶段对比测试 + 数据可视化
    ├── main/
    │   ├── a02yyuw.c    独立驱动（batch-read）
    │   ├── urm09.c      独立驱动（V3.0 寄存器）
    │   └── main.c       测试调度 + JSON 输出
    ├── tools/
    │   └── visualize.py  Python 可视化
    ├── 实验操作指南.md    详细操作手册
    └── ultrasonic_experiment_plan.md
```

调试工具验证硬件正常后，即可进入正式实验项目。

正式实验会自动执行 7 个测试阶段，**每阶段结束后输出统计数据并暂停**，在串口监视器中按 ENTER 后才继续下一阶段，方便在阶段间调整反射板位置和角度。

---

*适用于 ESP32-S3 + ESP-IDF v6.x*
